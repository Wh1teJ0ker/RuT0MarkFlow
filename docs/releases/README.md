# 版本文档索引

## 1. 目录定位
- `docs/releases/` 用于承接版本规划和变更记录。
- 根目录 `docs/01` 到 `docs/05` 保持为 MVP 基线文档。
- 每个版本固定维护两份文档：
  - `规划需求.md`：定义该版本的目标、范围、验收标准和不做项。
  - `更新日志.md`：记录该版本的实际实施状态、关键变更和验证结果。
- 跨版本的演进主线与节奏由 `docs/releases/路线规划.md` 统一约束。

## 2. 当前版本
- **当前版本：`v0.1.0`（融合基线版本，已完成验收）**
- `v0.1.0` 将 MVP 基线及全部稳定化、跨平台修复、自动更新、窗口修复、设置界面、圆角图标与 CI 优化的成果融合为单一版本。
- 后续版本方向待产品负责人确定后另行规划，当前不做路线规划。

## 3. 当前目录结构
```text
docs/releases/
  README.md
  路线规划.md
  v0.1.0/
    规划需求.md
    更新日志.md
```

## 4. 维护规则
- 每个版本实施阶段只更新该版本 `规划需求.md` 与 `更新日志.md`。
- 若新增版本，先补齐对应 `规划需求.md`，再开始代码和发布工作。
- 任何当前实施版本发生实际变更后，同步更新对应 `更新日志.md`。
- 若版本完成验收，再将结论回写 `docs/05-验收记录.md`。
- 任何主线方向调整，先更新 `路线规划.md`，再调整对应版本规划。
