# 后续版本路线规划

## 当前状态

**当前版本：`v0.1.0`（融合基线版本）**

`v0.1.0` 已完成 MVP 基线及全部稳定化、跨平台修复、自动更新、窗口修复、设置界面、圆角图标与 CI 优化的融合。

## 路线方针

**当前不做后续版本路线规划。**

后续版本的迭代方向由产品负责人（用户）给出建议和方向后，再行确定与规划。在此之前，不预设任何后续版本号、版本主题或版本范围。

## 新版本启动规则

当产品负责人确定下一个版本的迭代方向后：

1. 在本文件中补充新版本的方向声明。
2. 在 `docs/releases/<version>/` 下建立 `规划需求.md`。
3. 实施过程中更新对应 `更新日志.md`。
4. 版本完成后，验收结论回写 `docs/05-验收记录.md`。
