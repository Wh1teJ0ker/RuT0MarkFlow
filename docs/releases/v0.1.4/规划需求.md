# v0.1.4 规划需求

## 1. 版本信息
- 版本号：`v0.1.4`
- 类型：patch
- 状态：已完成验收

## 2. 目标
修复用户报告的核心问题并优化发布质量：(1) 修复 Mac/Windows 窗口无法关闭的根因（Rust 端无条件 prevent_close）；(2) 精简 Release 资产到每平台 1-2 个，便携版改为真正的免安装（skip 模式，依赖系统 WebView2）；(3) 图标从正方形优化为圆角风格；(4) 新增设置页面并迁移更新检查功能。全部验证命令保持绿色。

## 3. 范围

### 3.1 窗口关闭修复（T1）
- `src-tauri/src/lib.rs`：`on_window_event` 中的 `prevent_close()` 改为条件触发（仅当文档 dirty 时）
- `src-tauri/src/commands/state.rs`：新增 dirty state 同步命令
- `src/app/App.tsx`：document.isDirty 变化时同步到 Rust；destroy() 包 try/catch
- 新增 Rust 测试覆盖条件 prevent_close 逻辑

### 3.2 设置页面骨架（T2）
- `src/components/settings/SettingsPage.tsx`（新建）：独立页面组件，包含外观（主题切换）、关于（版本信息）、更新区块占位
- `src/components/toolbar/Toolbar.tsx`：主题切换按钮旁新增设置按钮
- `src/app/App.tsx`：新增 `isSettingsOpen` state + 条件渲染

### 3.3 更新检查迁移（T3）
- 更新 UI 从 StatusBar 迁移到设置页面的「更新」区块
- `src/services/updater/mod.ts`：错误信息优化，区分网络错误 / 404 / 无更新
- `src/components/statusbar/StatusBar.tsx`：移除更新相关 props 和 JSX

### 3.4 Release 精简 + 便携版免安装（T4）
- `src-tauri/tauri.conf.json`：bundle.targets 从 "all" 改为显式列表；webviewInstallMode 改为 skip（依赖系统 WebView2，免安装）
- `.github/workflows/release.yml`：matrix args 添加 --bundles 指定格式；命名 pattern 优化
- 资产精简：macOS 只 .dmg；Windows NSIS .exe + 便携版 .exe；Linux .deb + .AppImage

### 3.5 图标圆角优化（T6）
- 替换 `src-tauri/icons/app-icon-source.png` 为圆角设计源图（RGBA，四角透明）
- 从新源图重新生成所有派生图标（PNG / ICO / ICNS）

### 3.6 版本同步 + 文档收口（T5）
- 版本号 0.1.3 -> 0.1.4（5 个文件同步）
- `docs/releases/v0.1.4/` 文档更新
- `scripts/check-repo-hygiene.mjs` 版本断言更新

## 4. 不做项
- 不引入自建更新服务器（固定 GitHub Releases）
- 不引入全局状态管理库
- 不改动渲染管线（chunker / sanitize / marked / KaTeX 配置）
- 不改动主题系统核心逻辑（仅新增设置页面入口）
- 不改动 Rust 后端 updater 命令逻辑
- `v0.2.0` 相关文件不动
