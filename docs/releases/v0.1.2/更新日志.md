## 2. 关键变更

### CRITICAL-1：Tauri v2 capabilities + assetProtocol.scope
- 新增 `src-tauri/capabilities/default.json`，授予 `core:default`、`opener:default`、`dialog:default`
- 当前项目依赖的 Tauri 2 schema 中，`core:default` 已包含 `core:resources:default`；不存在 `core:asset:default` 这一权限名，因此 asset protocol 的访问边界由 `app.security.assetProtocol.scope` 控制
- `tauri.conf.json` 新增 `app.security.assetProtocol.scope: ["$APPDATA/**", "$RESOURCE/**", "$HOME/**"]`

### CRITICAL-2：Windows WebView2 离线安装
- `tauri.conf.json` 新增 `bundle.windows.webviewInstallMode: { type: "embedBootstrapper" }`

### LIKELY-1：文件编码——BOM 剥离 + GBK 回退
- `reader.rs` 改用 `std::fs::read` 读取原始字节，新增 `decode_content` 函数：
  - 先剥离 UTF-8 BOM（`\u{FEFF}`）
  - 尝试 UTF-8 解码（fast path）
  - 失败后通过 `encoding_rs::GBK.decode()` 回退（覆盖 GBK、GB2312）
  - 再失败用 `encoding_rs::GB18030.decode()` 回退
  - 全部失败返回明确编码错误
- 新增 6 条 Rust 测试覆盖 BOM 剥离、GBK 解码、BOM+GBK 组合、无效编码拒绝、纯 UTF-8 保持、BOM-only 空内容

### LIKELY-2：前端图片路径反斜杠归一化
- `src/services/render/resource.ts` 的 `resolveRelativePath` 在拼接前对 `rootPath`、`documentDir`、`src` 调用 `.replace(/\\/g, "/")`，确保 Windows 路径（如 `C:\Users\foo`）中的反斜杠被统一转换为正斜杠
- 新增 3 条 TS 测试覆盖纯反斜杠路径、混合分隔符路径、含 Unicode 目录名

### LIKELY-3：错误详情路径混合分隔符
- `src-tauri/src/commands/document.rs` 中 3 处 `format!("{}/{}", ...)` 路径拼接改为 `std::path::Path::new(&root_path).join(&relative_path)`

### LIKELY-4：CRLF 换行符归一化
- `src/services/render/index.ts` 的 `renderMarkdown` 函数入口新增 Step 0：`content.replace(/\r\n/g, "\n")`，确保所有下游（chunker、marked、KaTeX）接收一致 `\n` 换行
- 保存时 `writer.rs` 保持原样写入，不强制改写用户换行符
- 新增 2 条 TS 测试验证 `\r\n` 内容渲染后无残留 `\r`、chunker 保留原始内容

### MINOR-1：writer.rs 冗余 re-canonicalize 移除
- `write_to_any_path` 函数中 72-81 行的冗余 `canonicalize` + `strip_prefix` + `join` + `canonicalize` 代码块已移除，保留单次 `canonical.starts_with(&root)` 验证

### MINOR-2：indexer.rs 边界契约注释
- `build_index_tree` 文档注释增加路径分隔符假设说明，明确「入口必须经 `normalize_sep` 归一化」
