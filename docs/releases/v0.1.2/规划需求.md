# v0.1.2 规划需求

## 1. 版本信息
- 版本号：`v0.1.2`
- 类型：patch
- 状态：verified_complete

## 2. 目标
修复 Windows 平台无法正常打开应用、无法打开文档、图片无法加载、外链无法打开等问题，并清扫全仓库中硬编码路径分隔符与严格 UTF-8 假设，使 RuT0MarkFlow 在 macOS / Windows / Linux 三个平台上行为一致。

## 3. 范围

### 3.1 关键修复（CRITICAL）
- **CRITICAL-1**: 新增 `src-tauri/capabilities/default.json`，授予 `core:default`、`opener:default`、`dialog:default` 权限；当前 Tauri 2 schema 中 `core:default` 已包含 `core:resources:default`，不存在 `core:asset:default` 命名；`tauri.conf.json` 新增 `app.security.assetProtocol.scope`（`$APPDATA/**`、`$RESOURCE/**`、`$HOME/**`）
- **CRITICAL-2**: `tauri.conf.json` 新增 `bundle.windows.webviewInstallMode` → `embedBootstrapper`，确保 Windows 离线可用

### 3.2 文件编码修复
- `reader.rs`: 添加 UTF-8 BOM 剥离 + GBK/GB2312/GB18030 回退解码（`encoding_rs` 库）
- 新增 Rust 测试覆盖 BOM 剥离、GBK 解码、BOM+GBK 组合、无效编码拒绝

### 3.3 路径分隔符统一
- `document.rs`: 错误详情路径改用 `PathBuf::join`，不再 `format!("{}/{}", ...)`
- `resource.ts`: `resolveRelativePath` 归一化所有反斜杠为正斜杠
- `indexer.rs`: 增加边界契约注释，说明入口必须经 `normalize_sep` 归一化

### 3.4 CRLF 归一化
- `render/index.ts`: `renderMarkdown` 入口对内容做 `content.replace(/\r\n/g, "\n")`，确保 marked `breaks:true` 不受残留 `\r` 影响
- 保存时保持原样（`writer.rs` 不强制改写用户换行符）

### 3.5 代码清理
- `writer.rs`: 移除 72-81 行冗余 re-canonicalize 块（重复验证，Windows 上有 panic 风险）

### 3.6 版本与文档同步
- `package.json` / `Cargo.toml` / `tauri.conf.json` / `version-manifest.json` / `src/version.ts` → 0.1.2
- `docs/releases/v0.1.2/规划需求.md` / `docs/releases/v0.1.2/更新日志.md` 激活
- `docs/releases/README.md` 指向 v0.1.2
- `scripts/check-repo-hygiene.mjs` 更新 v0.1.2 断言

## 4. 不做项
- 不引入新功能（保持定位：本地化、最轻量）
- 不引入语法高亮、虚拟滚动、增量索引等重特性
- 不改动主题系统、状态栏、查找栏等已验证 UI
- 不改动 `.github/workflows/release.yml` 的 matrix / job 结构
- 不移除或改动 `pressure_bench.rs`
- `v0.1.3` 相关文件不动
