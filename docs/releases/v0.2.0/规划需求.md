# v0.2.0 规划需求

## 1. 版本定位
- 版本号：`v0.2.0`
- 版本类型：minor
- 主题：写作体验增强，把“能写”升级为“好写”。
- 上游依据：`docs/releases/路线规划.md` 第 3 节。
- 前置基线：`v0.1.1` 稳定化版本已验收通过。

## 2. 本版本要解决的问题
- MVP 阶段编辑器只保证基础文本输入和保存闭环，块元素操作仍不够顺手。
- 图片插入流程偏原生，缺少粘贴、拖拽和自动相对路径落盘。
- 无感渲染模式以阅读为主，轻编辑体验不足。
- 快捷键体系只覆盖全局操作，缺少 Markdown 语法级快捷操作。
- 文件打开方式单一，缺少拖拽入口。

## 3. 范围

### 3.1 无感渲染模式块级轻编辑
- 标题、段落、列表项、引用、任务项支持直接编辑。
- 代码块、表格、公式在 MVP 阶段允许源码回退，本版本优化回退体验。
- 块级编辑不要求做到完整所见即所得，优先保证高频块元素可用。

### 3.2 图片插入与相对路径落盘
- 支持剪贴板图片粘贴插入。
- 支持图片文件拖拽插入。
- 插入图片时自动落盘到工作区内可配置目录，并在文档中以相对路径引用。
- 支持配置图片落盘目录与命名规则。

### 3.3 表格、公式、代码块编辑体验
- 表格支持可视化插入行/列与基础编辑。
- 公式块支持源码编辑与实时预览。
- 代码块支持语言切换与保留缩进。

### 3.4 Markdown 语法快捷操作
- 加粗、斜体、删除线、行内代码的快捷键。
- 插入标题、列表、引用、表格、代码块、分隔线的快捷操作。
- 任务列表勾选切换快捷操作。

### 3.5 拖拽打开与导入
- 拖拽 Markdown 文件到窗口可直接打开。
- 拖拽图片到窗口可插入到当前位置。
- 拖拽行为需与当前未保存确认流程兼容。

## 4. 明确不做
- 全文搜索（留给 v0.3.0）。
- 文档大纲与锚点导航增强（留给 v0.3.0）。
- 导出能力（留给 v0.3.0）。
- AI 能力（留给 v0.4.0）。
- 多窗口与多标签（留给 v0.5.0）。
- 双向链接与知识图谱。
- 云同步与账号。

## 5. 设计约束
- 所有增强必须建立在 MVP 已有双模式、渲染和工作区索引之上，不重写底座。
- 图片落盘路径解析统一基于当前文档目录和工作区根路径，避免多处自行拼路径。
- 块级编辑结果与源码必须双向一致，不允许出现渲染态与源码态漂移。
- 快捷键不与全局快捷键冲突。
- 长文档性能不退化，超长文档场景下块级编辑不得引入持续卡顿。

## 6. 验收标准
- 无感渲染模式下可对标题、段落、列表、引用、任务项进行直接编辑。
- 图片粘贴与拖拽插入可用，并自动以相对路径落盘到工作区。
- 表格、公式、代码块编辑体验明显优于 MVP。
- Markdown 语法快捷操作可用且不与全局快捷键冲突。
- 拖拽 Markdown 文件可直接打开，并兼容未保存确认流程。
- 超长文档场景下编辑、滚动、模式切换不出现持续性卡顿。
- `pnpm test`、`pnpm build`、`cargo test --manifest-path src-tauri/Cargo.toml`、`cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets -- -D warnings` 全部通过。
