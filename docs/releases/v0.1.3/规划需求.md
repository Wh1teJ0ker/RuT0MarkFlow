# v0.1.3 规划需求

## 1. 版本信息
- 版本号：`v0.1.3`
- 类型：patch
- 状态：活跃实施中

## 2. 目标
为 RuT0MarkFlow 接入 Tauri v2 官方 updater 机制，实现「应用启动后自动检查 GitHub Releases 上的新版本，有更新时状态栏提示并可一键下载安装，用户也可手动触发检查」，全部跨平台可用。

## 3. 范围

### 3.1 后端 / Tauri 配置
- `Cargo.toml` 新增 `tauri-plugin-updater = "2"` 依赖
- `lib.rs` 注册 `tauri_plugin_updater::Builder::new().build()` 插件
- `commands/updater.rs`（新建）：`check_for_updates` / `install_update` 两个 `#[tauri::command]`
- `tauri.conf.json` 新增 `plugins.updater`（真实 pubkey、GitHub Releases endpoints、windows installMode passive）
- `capabilities/default.json` 追加 `updater:default` 权限

### 3.2 前端服务层
- `types/index.ts` 新增 `UpdateCheckResult` / `UpdateInstallResult` / `UpdateStatus` 类型；`AppErrorOperation` 追加 `check-update` / `install-update`
- `services/tauri/index.ts` 追加 `check-update` / `install-update` 错误标题和恢复标签
- `services/updater/mod.ts`（新建）：`checkForUpdates()` / `installUpdate()` async 函数

### 3.3 前端 UI
- `StatusBar.tsx` 新增「有新版本 vX.Y.Z」可点击指示器 +「检查更新」按钮
- `App.tsx` 新增启动期自动检查 effect（30s 超时、失败静默）、`handleCheckForUpdates` / `handleInstallUpdate` 回调

### 3.4 CI / 发布
- `.github/workflows/release.yml` 的 `uploadUpdaterJson` 改为 `true`，注入 `TAURI_SIGNING_PRIVATE_KEY` 与 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

## 4. 不做项
- 不引入自建更新服务器（固定 GitHub Releases）
- 不做增量/差分更新（整包替换）
- 不引入全局状态管理库
- 不改动 v0.1.2 已修复的跨平台路径/编码/CRLF 逻辑
- 不改动渲染管线、主题系统、查找栏、侧栏索引
- 不强制用户更新（可忽略提示）
- 不引入更新开关设置项
- `v0.2.0` / `v0.1.4` 相关文件不动
