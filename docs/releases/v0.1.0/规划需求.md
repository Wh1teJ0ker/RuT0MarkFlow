# v0.1.0 规划需求

## 1. 版本信息

- 版本号：`v0.1.0`
- 版本类型：基线版本（融合版）
- 对外发布日期：2026-07-14

## 2. 版本定位

`v0.1.0` 是 RuT0MarkFlow 的**首个对外发布版本**，融合了 MVP 基线及全部稳定化、跨平台修复、自动更新、窗口修复、设置界面与圆角图标等成果。

本版本将此前所有迭代版本（v0.1.1 稳定化、v0.1.2 跨平台修复、v0.1.3 自动更新、v0.1.4 窗口修复+设置+图标）的全部成果融合为单一基线版本，不再保留中间版本痕迹。

## 3. 目标

在一个版本内交付：

1. **MVP 核心闭环**：工作区驱动、双模式编辑/预览、Markdown 渲染（含公式）、相对资源解析、状态恢复，25 项验收标准全部通过。
2. **发布与诊断基础设施**：版本同步验证（5 文件一致性）、README 产品文档、Release 便携版资产、仓库规范门禁（repo:check）、压测门禁固化。
3. **跨平台一致性**：Tauri v2 capabilities 配置、文件编码 BOM/GBK/GB18030 兼容、路径分隔符统一、CRLF 规范化。
4. **GitHub 自动更新**：Tauri updater 集成、启动自动检查、设置页手动检查与一键更新。
5. **核心体验修复**：窗口关闭逻辑修复（仅脏文档拦截关闭）、设置页（外观/关于/更新）、圆角图标、便携版真正免安装（依赖系统 WebView2）。
6. **CI 编译加速**：rust-cache 缓存、LTO thin、并行 lint/rust-check、增量编译。

## 4. 范围

### 4.1 发布与版本同步
- 5 文件版本同步：`package.json`、`src-tauri/Cargo.toml`、`src-tauri/tauri.conf.json`、`version-manifest.json`、`src/version.ts`。
- `check-version-sync.mjs` 验证前端 fallback 常量一致性。
- `check-repo-hygiene.mjs` 仓库规范门禁（绝对路径、个人目录、stale tag、图标、release 配置）。

### 4.2 README 与 Release 资产
- README 补齐：产品简介、技术栈、本地开发、桌面构建、测试命令、Release 产物命名规则、Updater 签名密钥说明。
- Windows 独立便携版资产（`[name]_[version]_[platform]_[arch]_portable[ext]`）。
- macOS `.dmg`、Windows NSIS `.exe` + 便携版 `.exe`、Linux `.deb` + `.AppImage`。

### 4.3 跨平台修复
- **capabilities**：`core:default` + `opener:default` + `dialog:default` + `assetProtocol.scope`。
- **文件编码**：`reader.rs` BOM 剥离 → UTF-8 快速路径 → GBK → GB18030 回退 + 6 个 Rust 测试。
- **路径分隔符**：`resource.ts` 反斜杠→正斜杠规范化 + 3 个 TS 测试；`document.rs` PathBuf::join。
- **CRLF**：`render/index.ts` Step-0 `\r\n→\n` 规范化 + 2 个 TS 测试。
- **webviewInstallMode**：`skip` 模式——不嵌入/不下载 WebView2 安装程序，依赖系统已装 WebView2（Windows 10 1803+ / Windows 11 已预装），安装版和便携版均双击即用。

### 4.4 GitHub 自动更新
- 后端：`tauri-plugin-updater` 依赖、插件注册、`commands/updater.rs`（`check_for_updates` / `install_update`）、`tauri.conf.json` updater 配置（pubkey + endpoints + windows passive）、`updater:default` capability。
- 前端服务：`services/updater/mod.ts`（`checkForUpdates` / `installUpdate`），更新类型定义，错误分类（network / 404 / no-update）。
- 前端 UI：设置页更新检查入口（手动检查 + 一键下载安装），App 启动自动检查（30s 超时，静默失败）。

### 4.5 核心体验修复
- **窗口关闭**：`lib.rs` 条件 `prevent_close`（仅 `DocumentDirtyState` = dirty 时拦截关闭），前端脏状态同步，`destroy()` 包裹 try/catch。
- **设置页**：`SettingsPage.tsx`（外观/主题切换 + 关于/版本 + 更新检查），Toolbar 设置按钮入口。
- **圆角图标**：`app-icon-source.png` → 圆角设计（RGBA，四角透明，~22% 圆角半径），重新生成全部衍生图标（png/ico/icns 各尺寸）。

### 4.6 CI 编译优化
- `Swatinem/rust-cache@v2` 缓存 Cargo registry + target/。
- `lto = "thin"`（并行 LTO，编译快、性能损失 <5%）。
- release workflow 拆为 `lint`（version:check + repo:check + pnpm test + build）与 `rust-check`（cargo test + clippy + pressure:test）并行。
- `.cargo/config.toml` 启用 `incremental = true`。

### 4.7 全系统日志体系
- **Rust 后端**：`tauri-plugin-log` + `log` crate；`LogTarget::LogDir`（轮转文件）+ `LogTarget::Stdout`（dev 控制台）；release=Info, dev=Trace。
- **Mutex 中毒防护**：4 处 `.unwrap()` → match + `log::error!` + `into_inner()`，杜绝中毒 panic。
- **Rust 全面埋点**（9 个文件）：reader/writer/scanner/watcher/persistence/app_state/updater/document/workspace，纯补充层。
- **前端 logger**：`src/services/logger/index.ts`（级别分级、500 条环形缓冲、转发到 Rust）；`vite.config.ts` 注入 `__LOG_LEVEL__`；`main.tsx` 全局 error/unhandledrejection 捕获。
- **前端全面埋点**（7 个文件）：tauri/render/updater/workspace/document/App/ContentArea，替换所有 `console.*`。

### 4.8 文档修复与 README 简化
- README 从 151 行精简至 ~60 行，移出发布运维内容到新建 `docs/releases/releasing.md`。
- README 标题旁引用应用图标 `src-tauri/icons/128x128.png`。
- 核心文档（01-04）中所有 v0.1.1–v0.1.6、v0.2.0 版本引用统一为 v0.1.0 融合基线视角。
- QA 审计文档（00/01/04/06）全面重写，QA-014 和 QA-016 标记为 resolved。
