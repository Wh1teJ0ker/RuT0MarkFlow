# v0.1.0 更新日志

## 版本信息

- 版本号：`v0.1.0`
- 状态：`complete`
- 对外发布日期：2026-07-14

## 版本说明

`v0.1.0` 是 RuT0MarkFlow 的**首个对外发布基线版本**。本版本将 MVP 开发阶段及后续全部稳定化、跨平台修复、自动更新集成、窗口修复、设置界面、圆角图标与 CI 优化的成果融合为单一版本，不再保留中间迭代版本痕迹。

## 关键变更

### 1. 发布与版本同步

- **5 文件版本同步**：`package.json`、`src-tauri/Cargo.toml`、`src-tauri/tauri.conf.json`、`version-manifest.json`、`src/version.ts` 全部统一为 `0.1.0`。
- `check-version-sync.mjs` 验证前端 fallback 常量一致性。
- `check-repo-hygiene.mjs` 仓库规范门禁：绝对路径检测、个人目录检测、stale tag 链接检测、图标完整性检测、release 配置检测。
- `pnpm repo:check` 与 `pnpm pressure:test` 进入发布前 validate 流程。

### 2. README 与 Release 资产

- README 补齐：产品简介、技术栈、本地开发、桌面构建、测试命令、Release 产物命名规则、WebView2 运行时策略、Updater 签名密钥说明。
- Release 产物命名采用 Tauri 内置令牌模板：`[name]_[version]_[platform]_[arch]_[setup][ext]`。
- Windows 额外发布独立便携版：`[name]_[version]_[platform]_[arch]_portable[ext]`。
- 各平台产物：macOS `.dmg`、Windows NSIS `.exe` + 便携版 `.exe`、Linux `.deb` + `.AppImage`。

### 3. 跨平台修复

- **Tauri v2 capabilities**：`core:default` + `opener:default` + `dialog:default`；`assetProtocol.scope: [$APPDATA/**, $RESOURCE/**]`。
- **文件编码兼容**：`reader.rs` `decode_content` — BOM 剥离 → UTF-8 快速路径 → GBK → GB18030 回退，6 个 Rust 测试覆盖。
- **路径分隔符**：`resource.ts` `resolveRelativePath` 反斜杠→正斜杠规范化 + 3 个 TS 测试；`document.rs` 3 处路径拼接改用 `PathBuf::join`。
- **CRLF 规范化**：`render/index.ts` Step-0 `\r\n→\n`（写入保持用户换行风格）+ 2 个 TS 测试。
- **writer.rs 清理**：移除冗余 re-canonicalize 代码块。
- **indexer.rs**：补充边界契约注释。

### 4. GitHub 自动更新

- **后端**：`tauri-plugin-updater = "2"` 依赖；`lib.rs` 注册 updater 插件；新增 `commands/updater.rs`（`check_for_updates` 返回版本信息，`install_update` 下载+安装+重启）；`tauri.conf.json` 配置 `plugins.updater`（pubkey + endpoints + windows passive）；`updater:default` capability。
- **前端服务**：新增更新类型定义（`UpdateCheckResult` / `UpdateInstallResult` / `UpdateStatus`）；`services/tauri/index.ts` 错误标题/标签；新增 `services/updater/mod.ts`（`checkForUpdates` / `installUpdate`），错误分类（network / 404 / no-update）。
- **前端 UI**：更新检查入口迁移至设置页（手动检查按钮 + 一键下载安装）；App 启动自动检查（mount-once，30s 超时，静默失败）。
- **CI**：`release.yml` 注入 `TAURI_SIGNING_PRIVATE_KEY` + 密码；`uploadUpdaterJson=true`。

### 5. 核心体验修复

- **窗口关闭根因修复**：`lib.rs` 无条件 `prevent_close` → 条件 `prevent_close`（仅 `DocumentDirtyState` = dirty 时拦截关闭）；前端脏状态同步；`destroy()` 包裹 try/catch 确保监听器失败不阻断关闭。
- **设置页**：新增 `SettingsPage.tsx`（外观/主题切换 + 关于/版本信息 + 更新检查入口）；Toolbar 新增设置按钮；App 新增 `isSettingsOpen` 状态管理。
- **圆角图标**：`app-icon-source.png` → 圆角设计（RGBA，四角透明，~22% 圆角半径）；重新生成全部衍生图标（32x32、128x128、128x128@2x、icon.icns、icon.ico）。
- **WebView2 策略**：`webviewInstallMode` 从 `embedBootstrapper` → `skip`——不嵌入/不下载 WebView2 安装程序，依赖系统已装 WebView2，安装版和便携版均双击即用，真正免安装。

### 6. CI 编译加速

- **rust-cache**：`Swatinem/rust-cache@v2` 缓存 Cargo registry + `target/` 目录，命中时跳过 522 个 crate 重编译。
- **LTO 优化**：`lto=true` → `lto="thin"`（并行 LTO，编译速度接近无 LTO，运行时性能损失 <5%）；删除 `codegen-units=1`（恢复默认 16 并行代码生成）。
- **并行化**：release workflow 从 `validate → publish` 串行拆为 `lint`（version:check + repo:check + pnpm test + build）与 `rust-check`（cargo test + clippy + pressure:test）两个并行 job；publish 仅 `needs: [lint]`。
- **增量编译**：`src-tauri/.cargo/config.toml` 启用 `incremental = true`。
- **CI workflow 排除 tag push**：tag push 只触发 Release workflow，不重复跑 CI。
- **预期效果**：首次 release ~20min（LTO 优化），二次 release ~12-15min（缓存命中 + 并行）。

### 7. 全系统日志体系

为整个系统加入完整的日志系统，方便定位错误，覆盖 Rust 后端与 React 前端。

- **Rust 后端日志基础设施**：
  - 引入 `tauri-plugin-log = "2"` + `log = "0.4"`，与现有 updater/dialog/opener 插件模式一致。
  - `lib.rs` 注册 log 插件（`LogTarget::LogDir` 写入 `app_data_dir/logs/` 轮转文件 + `LogTarget::Stdout` dev 控制台）；release 级别 `Info`，dev 级别 `Trace`。
  - **Windows release 诊断必需**：`windows_subsystem = "windows"` 下 stderr 不可见，文件日志是 Windows release 构建诊断的唯一通道。
  - `capabilities/default.json` 添加 `log:default` 权限。
  - 4 处 Mutex `.unwrap()` 加中毒防护（`match .lock() { Ok → ..., Err(e) → log::error! + e.into_inner() }`），杜绝中毒 panic。
  - `health.rs` `.expect()` 改为 `match` + `log::error!` + 优雅降级。

- **Rust 后端全面埋点**（9 个文件，纯补充层，不改签名/返回/控制流）：
  - `document/reader.rs`：文件打开、编码检测（UTF-8/GBK/GB18030 回退）、路径遍历拒绝、大小超限。
  - `document/writer.rs`：原子写入流程、临时文件清理失败、symlink 拒绝。
  - `workspace/scanner.rs`：扫描开始、`read_dir` 失败、扫描完成。
  - `workspace/watcher.rs`：启动/停止、notify 错误、emit 失败、工作区消失。
  - `settings/persistence.rs`：加载失败、保存失败、JSON 损坏重置。
  - `app_state.rs`：授权、token 签发与消费、token 过期。
  - `commands/updater.rs`：检查开始、版本发现、下载进度、安装/重启。
  - `commands/document.rs`：打开/保存/另存为/取消。
  - `commands/workspace.rs`：选择/加载/刷新/watcher 生命周期。

- **前端 logger 模块**：
  - 新建 `src/services/logger/index.ts`：级别分级（trace/debug/info/warn/error）、500 条环形缓冲、console 输出（dev）、转发到 Rust 后端（动态 `import("@tauri-apps/api/core")` 避免 vitest 环境加载 Tauri API）。

- **前端全面埋点**（7 个文件，替换所有 `console.*`）：
  - `services/tauri/index.ts`：`invokeTauriCommand` 请求日志（命令名、延迟、成功/失败），删除原 TODO。
  - `services/render/index.ts`：替换 3 处 `console.warn` → `logger.warn`；渲染管线阶段日志。
  - `services/updater/mod.ts`：更新检查/安装生命周期日志。

## 功能概览（使用体验）

### 核心功能
- **工作区驱动**：选择本地文件夹后自动扫描 Markdown 文件，按目录层级建立索引树。
- **双模式体验**：无感渲染模式（沉浸式阅读）与双栏编辑模式（左编辑右预览），一键切换。
- **本地内容友好**：相对路径图片、相对链接、工作区内 Markdown 跳转、KaTeX 行内/块级公式渲染。
- **状态恢复**：自动恢复最近工作区、最近文档、最近模式，恢复失败安全降级到空工作区。
- **文件监听**：工作区内文件变化自动刷新索引（notify + 300ms 去抖）。
- **超长文档**：分段渲染 + `content-visibility: auto` 虚拟化，10000 行文档首屏即时显示。
- **错误隔离**：图片/公式加载失败不阻塞整篇渲染。

### 体验亮点
- **窗口关闭修复**：只有未保存的文档才会拦截关闭并提示用户；已保存的文档正常关闭。
- **设置页**：外观主题切换（深色/浅色）、关于/版本信息、更新检查入口（手动检查 + 一键下载安装）。
- **圆角图标**：四角透明圆角设计，符合现代桌面应用主流视觉风格。
- **便携版免安装**：Windows 便携版 `.exe` 双击即用，不嵌入 WebView2 安装程序，依赖系统已装运行时。
- **自动更新**：应用启动后自动检查 GitHub Releases 新版本，设置页可手动触发检查与更新。
- **全系统日志**：Rust 后端写入 `app_data_dir/logs/` 轮转文件（Windows release 诊断必需），前端 logger 转发到后端；全局 error/unhandledrejection 捕获；所有静默吞错路径补 warn 日志。
