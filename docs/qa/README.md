# QA 审计报告索引

## 1. 审计范围

本目录记录对当前 RuT0MarkFlow 完整代码、正式文档、配置、测试、CI 和发布链路的证据化审计。审计以发现问题和建立整改优先级为目标，本轮不修改业务代码。

## 2. 报告目录

1. `00-审计清单.md`：目标、范围、证据矩阵和严重度定义。
2. `01-文档与实现一致性审计.md`：版本、需求、交互、技术接口和验收记录差异。
3. `02-前端代码审计.md`：前端正确性、安全、性能、可访问性和测试盲区。
4. `03-Rust与Tauri审计.md`：命令授权、文件系统可靠性、Tauri 安全配置和跨平台风险。
5. `04-测试构建与发布审计.md`：测试、CI、供应链、版本、Release 和门禁问题。
6. `05-验证记录.md`：实际执行命令与结果。
7. `06-问题总表与整改优先级.md`：跨报告去重后的整改入口。

## 3. 总体结论

本轮审计为本地仓库静态审计与现有自动化门禁复核。所有核心测试、构建、Clippy、版本同步、仓库规范和 1000 文件压力测试通过，但静态审计发现多项现有门禁未捕获的高风险问题。

本轮审计边界：本地仓库静态审计 + 现有自动化门禁复核已完成。跨平台构建、远端 GitHub Release、真实 updater 升级链、依赖漏洞数据库扫描、GUI 人工验收和动态攻击验证未完成，已列为外部环境待验证项。

首要风险类别：

- 文件系统授权边界：Rust 后端信任前端传入的 root 和另存为 target path。
- 数据可靠性：保存非原子，且前端保存/另存为竞态可导致内容丢失或错误显示已保存。
- WebView 安全：CSP 禁用、asset scope 覆盖 HOME、图片错误处理存在二次 HTML 注入面。
- 工作区正确性：取消 picker 清空现有状态，相对 Markdown 链接解析错误。
- 发布供应链：Actions 未固定 SHA，Release 产物可早于 Rust 门禁上传。
- 文档可信度：v0.1.0 版本融合后各文档已统一口径；审计时发现的漂移项已通过版本重置修正。

## 4. 阅读建议

- 产品与项目负责人先读 `06` 和 `01`。
- 前端开发先读 `02`。
- Rust/Tauri 开发先读 `03`。
- 发布维护者先读 `04` 和 `05`。

## 5. 重要说明

- 本轮审计为本地静态审计与现有自动化门禁复核，不包含跨平台构建、远端发布环境、真实 updater 升级链和依赖漏洞数据库扫描。这些已列为外部环境待验证项，详见各报告未验证项和 `06` 第 8 节。
- “测试全部通过”不等于审计无问题；多个高风险场景当前没有对应测试。
- 严重度和优先级是两个独立维度：Critical 问题必然 P0，但 High 问题也可能是 P0（如数据丢失竞态）。
- 安全问题建议在公开发布前先修授权、原子写入、CSP/asset scope 和二次 HTML 注入。
- 本目录是审计快照；修复后应逐条更新状态、修复证据和回归命令。
