## 5. 低优先级文档与可维护性项

以下为文档漂移、可访问性和可维护性问题的汇总，未逐一拆分 ID，但建议在 P1/P2 整改过程中顺带修复：

- 验收记录行号大量失效（`docs/05-验收记录.md` 多处）。
- 测试计数已更新（245/32）。
- 更新日志标题已规范。
- 规划文档引用已清理的临时 HANDOFF。
- repo hygiene 能力被文档夸大。
- README 对本地构建签名要求表述过度。
- 未保存对话框缺 focus trap 和 aria 关联。
- 文件树 ARIA 语义不完整。
- 状态栏重试仅支持鼠标。
- 编辑器与查找输入缺标签。
- HTML 净化允许范围过宽。
- 外部链接协议和失败处理不完整。
- 设置持久化前端侧静默忽略错误。
- 生产主包超过 500 kB 警戒线。
- 依赖维护积压和弃用类型包。
- 错误码契约不统一。
- 编码回退可能误判并不可逆转码。
- 时间戳秒精度且 DefaultHasher 非稳定协议。
- reader.rs 超大文件测试名不符实（`reader.rs:217-227` 未创建超过 10 MB 文件）。
- repo hygiene 硬编码历史版本路径。

## 7. 修复完成定义

每项整改关闭前至少需要：

- 代码或文档修改已经落盘。
- 对应缺陷有可重复的回归测试；纯文档问题有代码/配置交叉核对证据。
- `pnpm test`、`pnpm build`、`cargo test --locked`、`cargo clippy --locked --all-targets -- -D warnings` 通过。
- 涉及安全边界时增加负向测试，而不只测试正常路径。
- 涉及 CI/Release 时对 workflow 权限、依赖和 job DAG 做静态验证，必要时在测试 tag 上做真实 dry run。
- 在本表中记录修复 commit、验证命令、日期和最终状态。
