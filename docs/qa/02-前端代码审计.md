# 前端代码审计

## 1. 审计结论

前端测试和构建当前通过，但静态审计发现多项现有测试未覆盖的竞态、正确性和安全问题。

发现：
- 高：5 项
- 中：10 项
- 低：5 项

## 2. 高严重度问题

### FE-H1：普通保存期间继续输入会被误标为已保存

位置：`src/app/App.tsx:430-446`

保存请求发送旧快照，成功回调却把最新 `prev.content` 写入 `lastSavedContent` 并无条件清除 dirty。磁盘只保存 A，界面存在 A+B，但状态显示已保存，关闭时 B 可丢失。

建议：捕获 `contentToSave` 与文档 ID；成功后设置 `lastSavedContent=contentToSave`，并以最新内容比较 dirty；丢弃过期文档的保存响应。

### FE-H2：另存为期间继续输入会被旧快照直接覆盖

位置：`src/app/App.tsx:489-509`

成功回调使用闭包中的旧 `document.content` 整体替换状态，保存期间新增输入会立即从编辑器消失。

建议：函数式 setState，保留 `prev.content`，仅更新路径和已保存快照；增加操作 ID。

### FE-H3：取消工作区选择会清空现有工作区和文档（已修复）

位置：
- `src/modules/workspace/mod.ts`
- `src/app/App.tsx`

历史问题：系统 picker 取消被映射为 idle/null，App 不区分取消与成功切换，直接清空状态。若用户先选择“不保存并继续”，随后取消 picker，未保存内容仍会丢失。

现状：`WorkspaceSelectionResult` 新增显式 `cancelled` 标志；`selectWorkspace` 在 `CANCELLED` 分支置 `cancelled: true`；`handleSelectWorkspace` 在收到 `cancelled` 时恢复 picker 打开前快照的工作区/文档状态，仅在成功选择并加载新工作区后才替换旧上下文。

### FE-H4：图片错误占位使用 outerHTML，形成二次 HTML 注入面

位置：`src/components/content/ContentArea.tsx:73-80`

DOMPurify 后又把解码后的 `src/alt` 拼入 `outerHTML`，新 HTML 不再净化；CSP 同时被禁用。

建议：使用 `createElement`、`textContent`、`dataset` 与 `replaceWith`，禁止字符串 HTML；补恶意属性实体测试。

### FE-H5：嵌套目录 Markdown 相对链接解析错误

位置：
- `src/services/render/resource.ts:80-89`
- `src/components/content/ContentArea.tsx:105-112`

`docs/a.md` 中的 `b.md` 被当作工作区根目录 `b.md`，未基于当前文档目录规范化；fragment 也会丢失。

建议：统一解析 `documentDir + href`，规范化 `.`/`..`，保留 fragment，拒绝工作区逃逸。

## 3. 中严重度问题

### FE-M1：图片路径可通过 `../` 逃逸工作区
位置：`src/services/render/resource.ts:113-127`；`src-tauri/tauri.conf.json:23-25`。字符串拼接不消解 `..`，asset scope 又允许 `$HOME/**`。应由 Rust canonicalize 并限制当前授权工作区。

### FE-M2：CSP 完全关闭
位置：`src-tauri/tauri.conf.json:21-22`。当前依赖 DOMPurify，但存在 dangerouslySetInnerHTML、outerHTML 和外部资源。一旦净化绕过便缺少纵深防御。

### FE-M3：并发未保存确认会覆盖 resolver
位置：`src/app/App.tsx:65-80` 及多个调用点。只有一个 `resolveConfirmRef`，后续请求可让前一 Promise 永久悬挂或执行错动作。建议互斥/队列。

### FE-M4：超长文档最终仍创建完整 DOM，不是真虚拟化
位置：`src/modules/render/mod.ts:178-247`、`ContentArea.tsx:293-320`、`FindBar.tsx:38-95`。分批仅推迟成本，反复拼接 HTML 并重建完整 DOM；查找同步遍历和 mark 全量匹配。

### FE-M5：独立 chunk 渲染改变跨块语义
位置：`src/services/render/chunker.ts:36-124`、`src/modules/render/mod.ts:180-230`。未完整保护 `~~~`、列表、表格、引用、HTML block、引用定义等。建议先解析 AST，再按顶层 block 分块。

### FE-M6：索引刷新缺少请求序列控制
位置：`src/app/App.tsx:322-339,407-415`。慢旧请求可覆盖新结果。建议 latest-request-wins、root 二次核对和事件合并。

### FE-M7：未保存对话框没有真正 focus trap
位置：`src/components/dialogs/UnsavedConfirmDialog.tsx:17-29,46-96`。Tab 可逃出、不恢复焦点、Enter 被全局强制为保存、缺 aria 关联。

### FE-M8：文件树 ARIA 语义不完整
位置：`src/components/sidebar/IndexTree.tsx:21-35,76-151`。缺 role=tree/group、aria-expanded/selected、roving tabindex 和方向键导航。

### FE-M9：状态栏保存重试仅支持鼠标
位置：`src/components/statusbar/StatusBar.tsx:74-86`。动态 role=button 但无 tabindex 和键盘行为。应使用原生 button。

### FE-M10：编辑器与查找输入缺少明确标签
位置：`ContentArea.tsx:240-245,293-302`、`FindBar.tsx:164-187`。应增加 label/aria-label 和结果 live region。

## 4. 低严重度问题

### FE-L1：32 位非加密 hash 作为缓存唯一键
位置：`src/modules/render/mod.ts:45-59,135-144`。碰撞时可能显示错误内容。建议命中后比较原文或使用更宽稳定 hash。

### FE-L2：HTML 净化允许范围过宽
位置：`src/services/render/sanitize.ts:39-58`。允许任意 style、所有 data-* 和较广 SVG 属性，可能造成 UI 仿冒、遮挡和性能攻击。

### FE-L3：外部链接协议和失败处理不完整
位置：`ContentArea.tsx:115-125`、`resource.ts:75-97`。openUrl Promise 未捕获，大小写/未知 scheme 可能落入 WebView 默认导航。

### FE-L4：设置持久化失败被静默忽略且可能乱序
位置：`src/modules/workspace/mod.ts:179-186`、`src/app/App.tsx:290-300`。建议 debounce、串行化、版本号和失败诊断。

### FE-L5：生产主包超过默认警戒线
本次构建主 JS 为 553.26 kB（gzip 169.12 kB），建议懒加载 KaTeX、设置页和 updater，或配置 manualChunks。

- 保存/另存为期间继续输入、双保存乱序、保存期间切换文档。
- 已有工作区时取消 picker，以及丢弃确认后取消 picker。
- 并发未保存确认、重复窗口关闭、连续快捷键。
- 图片错误后的二次 DOM 注入和 style/SVG 复杂载荷。
- 嵌套相对链接、fragment、Unicode、Windows path、图片穿越/symlink。
- 真实 10 MB 长文档、chunk 跨列表/表格/blockquote/HTML/fence 语义。
- 高密度查找结果、内存上限和性能 profiler。
- axe、focus trap、tree 键盘模式和 live region。
- CSP、asset protocol 和 opener 实际运行测试。

## 6. 良好项

- Markdown 初次注入前经过 DOMPurify。
- 文档打开实现 latest-request-wins，且已有乱序测试。
- 窗口关闭有 Rust/前端 dirty-state 协作。
- 新建、打开、关闭、切换工作区基本接入未保存确认。
- 渲染有 250ms debounce、分块调度和缓存。
- 图片、公式和渲染失败有降级反馈。
- 编辑历史有容量限制。
- 本次 27 个测试文件、196 个测试通过；构建通过。
